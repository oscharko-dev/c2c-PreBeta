# c2c-bff W0.2 API contract (Issue #172)

The BFF is the only API surface the Studio is allowed to call. W0.2 extends the
W0/W0.1 surface with the product-level signals the UI needs to drive the
agent-backed transformation loop without leaking provider credentials,
internal service topology, or raw orchestrator stack traces.

The canonical schema is `services/c2c-bff/openapi.yaml`. This document
summarises the W0.2-specific behaviour and is the entry point reviewers should
read when validating the contract.

## Run lifecycle

1. The Studio submits a transform via `POST /api/v0/transform`.
2. The Studio polls `GET /api/v0/runs/{runId}` for the lightweight run summary.
3. The Studio reads `GET /api/v0/runs/{runId}/workflow` for the structured W0.2
   contract (state machine, active agent, repair budget, failure code).
4. The Studio reads artifact endpoints (`/generated`, `/generated/files`,
   `/generated/files/{path}`, `/build-test`, `/evidence`, `/artifacts`,
   `/progress`, `/learning`, `/events`) as it builds the run view.

## `POST /api/v0/transform`

Required body fields:

- `sourceText` (string) — COBOL source the user submitted.

Optional body fields:

- `programId` — explicit program id. If absent the BFF derives one from the
  source text (PROGRAM-ID directive or sha256 prefix).
- `sourceName` — short label shown back on the run summary.
- `targetLanguage` — defaults to `java`. W0.2 rejects any other value with
  HTTP 400.
- `expectedOutput` — optional expected golden-master output. Forwarded to the
  orchestrator as `inputRef.expectedOutput`; the orchestrator pulls it into
  the COBOL-runtime oracle payload at the build/test runner step and at the
  transformation-agent invocation, so the verification/repair agent can
  compare against the exact bytes the user supplied. Empty string is
  treated as "not supplied" and falls back to deterministic defaults.
- `oracleInput` — optional structured input for oracle-mode comparison.
  Forwarded as `inputRef.oracleInput` (same handling as `expectedOutput`).
- `options` — opaque object passed through to the orchestrator.

Failure modes:

- 400 — validation error or programId is a reference program with
  `supportedInProductMode=false`.
- 413 — `sourceText` exceeds `C2C_TRANSFORM_SOURCE_MAX_BYTES`.
- 502 — orchestrator rejected the request. Response body carries
  `failureCode: "service_unavailable"`. The text never contains the
  orchestrator URL, provider keys, file system paths, or stack frames.
- 503 — orchestrator not configured (`C2C_ORCHESTRATOR_URL` unset).

## `GET /api/v0/runs/{runId}` — run summary

The run summary always exposes the W0.2 contract surface:

- `activeStep` — orchestrator step name or `null`.
- `agentAttemptCount` — total agent invocations so far (defaults to `0`).
- `repairBudget` — `{ limit, used, remaining }` (W0.2 bounds: 1..3) or `null`
  if the run has not produced a contract yet.
- `finalClassification` — one of `success`, `blocked`, `failed`, `cancelled`,
  `incomplete`, or `null` while the run is still active.
- `failureCode` — `W02UiErrorCode` or `null`. Orchestrator codes outside the
  closed enum collapse to `internal_error`. Connectivity failures map to
  `service_unavailable`.
- `failureMessage` — sanitized message or `null`. Stack traces, absolute file
  paths, URLs, API keys, and bearer tokens are stripped.

## `GET /api/v0/runs/{runId}/workflow` — full W0.2 contract

Response shape (`RunWorkflowView`):

- `state` — current W0.2 state machine state.
- `activeStep`, `activeAgent` — `activeAgent` is derived from `activeStep`
  and lands in a closed enum: `transformation_agent`,
  `verification_repair_agent`, `cobol_parser`, `semantic_ir`,
  `java_generator`, `build_test_runner`, `evidence_service`, or `null`.
- `agentAttemptCount`, `repairBudget`.
- `repairAttempts` — each attempt exposes `attemptNumber`, `repairDecision`,
  `failureCategory`, and boolean breadcrumbs (`hasModelInvocation`,
  `hasRepairInput`, `hasJavaCandidate`) plus an optional sanitized
  `rationale`. The raw `modelInvocationRef`, `repairInputRef`,
  `javaCandidateRef` URIs are deliberately omitted.
- `finalClassification`, `failureCode`, `failureMessage` — same semantics as
  on the run summary.
- `generatedJavaRef`, `buildTestResultRef`, `evidencePackRef` — each one is
  either `null` or `{ sha256, byteSize, kind }`. Internal `uri` and absolute
  `path` fields are intentionally not exposed.

The `source` field reports where the snapshot came from: `live`
(orchestrator's in-memory contract), `cached` (persisted
`w02-run-contract.json`), or `unavailable` (orchestrator unreachable or run
not yet accepted).

## Failure-code mapping

The BFF translates the orchestrator's `failureCode` to a closed
`W02UiErrorCode` enum. The mapping is:

| Orchestrator code | UI-safe code |
|---|---|
| `unsupported_cobol` | `unsupported_cobol` |
| `parse_failed` | `parse_failed` |
| `semantic_ir_failed` | `semantic_ir_failed` |
| `model_gateway_unavailable` | `model_gateway_unavailable` |
| `model_policy_denied` | `model_policy_denied` |
| `agent_timeout` | `agent_timeout` |
| `agent_contract_invalid` | `agent_contract_invalid` |
| `java_generation_failed` | `java_generation_failed` |
| `java_compile_failed` | `java_compile_failed` |
| `java_runtime_failed` | `java_runtime_failed` |
| `oracle_mismatch` | `oracle_mismatch` |
| `evidence_incomplete` | `evidence_incomplete` |
| `cancelled` | `cancelled` |
| _anything else_ | `internal_error` |
| _upstream unreachable_ | `service_unavailable` |

A run that is marked `blocked` / `failed` / `cancelled` but carries no
canonical code is reported as `internal_error` rather than silently dropped.

## Artifact size limits

`GET /api/v0/runs/{runId}/generated/files/{filePath}` enforces
`C2C_ARTIFACT_CONTENT_MAX_BYTES` (default: 1 MiB) in three independent layers
so that an oversized or malicious upstream cannot pin the BFF process or the
browser tab:

1. **Declared content-length pre-check.** The HTTP client rejects the
   request before reading any body bytes if the upstream's `content-length`
   header exceeds the cap. Failure surfaces as
   `UpstreamResponseTooLargeError`.
2. **Streaming cutoff.** As the body is streamed, the client tracks
   cumulative bytes received and aborts the request as soon as the total
   crosses the cap. The resolved response is marked `truncated: true` and
   the body is dropped.
3. **Envelope re-check.** After the JSON envelope is parsed, the handler
   takes the **maximum** of the upstream-declared `byteSize` and the
   measured byte length of the `content` string. A liar upstream that
   reports `byteSize: 1` while sending megabytes of `content` still
   fails this check.

The BFF returns HTTP 413 with:

```json
{
  "error": "artifact_too_large",
  "path": "src/main/java/c2c/HELLO01.java",
  "byteSize": 2097152,
  "limit": 1048576
}
```

`byteSize` may be absent when the streaming-cap path aborted before the
upstream supplied a measurable size.

Other text artifact endpoints (`/generated`, `/build-test`, `/evidence`,
`/progress`, `/learning`, `/events`, `/artifacts`) report a normalized
`incomplete` response when the orchestrator has not yet persisted the
artifact, so the UI can always render a stable shape.

## Environment

The W0.2 contract is driven by these environment variables:

- `C2C_ORCHESTRATOR_URL` — required for product mode. When unset the BFF
  fails closed and only honours diagnostic fixtures behind
  `C2C_ENABLE_DIAGNOSTIC_FIXTURES`.
- `C2C_EVIDENCE_URL`, `C2C_EXPERIENCE_LEARNING_URL`, `C2C_MODEL_GATEWAY_URL`,
  `C2C_HARNESS_URL` — optional upstreams. The BFF gracefully reports them as
  unavailable when unset.
- `C2C_TRANSFORM_SOURCE_MAX_BYTES` — body limit on `POST /api/v0/transform`
  (default: 1 000 000 bytes).
- `C2C_ARTIFACT_CONTENT_MAX_BYTES` — per-file size limit for the
  generated-file content endpoint (default: 1 048 576 bytes).
- `C2C_UPSTREAM_TIMEOUT_MS` — single upstream request timeout
  (default: 4 000 ms).

No secret is read or returned by the BFF; provider credentials live behind
the Model Gateway and never leave it.
