# W0.2 Epic #164 Closure Evidence

Issue [#164](https://github.com/oscharko-dev/c2c-PreBeta/issues/164)
is the Wave 0.2 parent epic for the first productive AI-assisted COBOL-to-Java
transformation loop: an orchestrator-controlled agent workflow on the
Experience Learning Harness, governed model access through the Model Gateway,
a bounded verification/repair loop, and the same deterministic build/test/
evidence gate that already protects the W0 and W0.1 paths.

This document records the closure evidence for the epic. It does not replace
the release gate in [`w0-2-release-gate.md`](w0-2-release-gate.md); it ties
the epic's child work back to that gate so reviewers can re-derive the result
from a clean checkout. It complements the W0 baseline gate
[`w0-release-gate.md`](w0-release-gate.md), the W0 corrective-path evidence
[`w0-corrective-epic-86.md`](w0-corrective-epic-86.md), and the W0.1 Studio
evidence [`w0-studio-epic-118.md`](w0-studio-epic-118.md).

## Closure Decision

The Issue #164 W0.2 epic is satisfied when the current `dev` branch provides,
per the W0.2 Definition of Done:

- a transformation run created through the BFF for a small supported COBOL
  program submitted from the Next.js Studio;
- an Orchestrator that controls the agent workflow on Harness-provided
  infrastructure, with the Harness observing and recording activity but not
  deciding workflow order;
- at least one productive model invocation through the Model Gateway in the
  Foundry-backed configuration, and a `modelInvocations[*].status == "skipped"`
  policy-decision entry in the deterministic configuration;
- a generated Java artifact persisted under the run artifact contract with a
  sha256 digest that matches across the `GeneratedView`, `BuildTestView`, and
  `EvidenceView` surfaces;
- a build/test result produced by the real `cobc`/`cobcrun` oracle through the
  build/test runner, not a synthetic shortcut;
- a behavioural comparison against the W0.2 oracle contract with
  `oracleComparison.matched == true` and `actualSha256 == expectedSha256`;
- an Evidence Pack carrying source COBOL, semantic IR reference, model
  invocation metadata, agent trajectories with an `orchestrator` role entry,
  build/test result, generated Java artifact references, and Harness events,
  with `completenessStatus == "complete"` and `status == "complete"`;
- a Studio UI that distinguishes success, blocked, failed, and incomplete runs
  explicitly, including the unsupported-source path that yields no generated
  Java artifact and no "Verified" affordance;
- first read-only Experience Learning signals for tool/capability availability,
  model invocation outcome, agent handoff, repair-loop progress, and
  generated-Java candidate outcome, without giving the Harness any workflow
  decision authority;
- a scripted W0.2 release gate that fails closed if any of the above is missing
  and is invoked on every PR by CI in its deterministic configuration.

Per the epic's Closing Rule, the integrated W0.2-11 release gate
([#175](https://github.com/oscharko-dev/c2c-PreBeta/issues/175)) is the
gatekeeper. It is closed and merged on `dev` via
[PR #190](https://github.com/oscharko-dev/c2c-PreBeta/pull/190); the
deterministic gate is wired into CI in
[`.github/workflows/ci.yml`](../../.github/workflows/ci.yml) and runs on every
PR; the Foundry-backed gate is documented for manual developer runs.

## Non-Scope (explicitly not claimed)

W0.2 closure does not claim, and must not be claimed in marketing, sales, or
release notes, the following items. They are out of scope for W0.2 and remain
on the longer roadmap, mirroring the epic's "Explicit Non-Scope for W0.2"
section and the release gate's [non-scope section](w0-2-release-gate.md#w02-non-scope-must-not-be-claimed):

- Full COBOL coverage beyond the W0.2 supported subset declared in
  [`docs/corpus/w02-acceptance.md`](../corpus/w02-acceptance.md).
- Production-grade customer migrations, customer onboarding, multi-tenant
  authentication, or any claim of production readiness for arbitrary customer
  source.
- A productive Experience Learning **decisioning** system. W0.2 ships only the
  first read-only signals; the Harness must not take workflow control
  decisions.
- Autonomous enterprise-level test generation.
- Complex mainframe semantics: file I/O, hierarchical databases, CICS, IMS,
  JCL chains, sort/merge utilities, EXEC SQL, EXEC CICS, VSAM, and
  copybook-heavy systems.
- Multiple parallel agent teams or multiple orchestrators. W0.2 ships one
  Orchestrator and one Transformation Agent plus one Verification/Repair Agent
  in a bounded repair loop.
- Target languages other than Java. W0.2 rejects any `targetLanguage` other
  than `java` at the BFF layer.

The W0 release-gate limitations continue to apply on top of the above.

## Re-evidencing this Epic

A reviewer can re-derive the closure decision from a clean checkout of `dev`
by running the same commands that re-evidence the W0.2 release gate. The
authoritative recipe lives in
[`docs/release/w0-2-release-gate.md`](w0-2-release-gate.md#re-evidencing-this-gate)
and [`docs/showcase/w0-2-reference-runbook.md`](../showcase/w0-2-reference-runbook.md);
the short form is:

```bash
# 1. Repository-level CI mirror
./scripts/ci-checks.sh

# 2. W0 baseline gate (must still pass — W0.2 does not weaken it)
W0_REFERENCE_RUN_ENV_FILE="$PWD/.env" \
  W0_REFERENCE_RUN_MODEL_GATEWAY_ENABLED=false \
  ./scripts/w0-reference-run.sh
C2C_LOCAL_ENV_FILE="$PWD/.env" \
  C2C_LOCAL_MODEL_GATEWAY_ENABLED=false \
  ./scripts/smoke-test-c2c-local.sh

# 3. W0.2 release gate (deterministic / no-Foundry). CI runs this on every PR.
C2C_LOCAL_ENV_FILE="$PWD/.env" \
  ./scripts/w0-2-release-gate.sh

# 4. Browser acceptance — both the W0.1 baseline and the W0.2 fixtures
cd apps/c2c-studio && \
  CI=1 \
  C2C_LOCAL_ENV_FILE="$PWD/../../.env" \
  C2C_LOCAL_MODEL_GATEWAY_ENABLED=false \
  npm run test:e2e:ci
```

To re-derive the productive agentic loop (the row that the deterministic gate
intentionally skips because public CI has no Foundry secrets), on a
workstation that has Microsoft Foundry credentials exported:

```bash
export AZURE_FOUNDRY_ENDPOINT=...    # see .env.example
export AZURE_FOUNDRY_API_KEY_REF=keyring/foundry/API_KEY
./scripts/w0-2-release-gate.sh --foundry
```

That command refuses to start if the Foundry secrets are absent. The CI
workflow declines to run it.

## Scope Boundaries

The #164 closure is a production-shaped W0.2 implementation of the agentic
COBOL-to-Java loop, not a claim that c2c is ready for arbitrary customer
production workloads. The deterministic W0 kernel — generated Java artifact,
compile, run, behavioural check, and complete evidence — remains the
gatekeeper for release. The agentic loop is additive: it provides the first
productive AI assistance, but it does not bypass the deterministic gate.

Per the [c2c Fachkonzept](../concept/c2c-fachkonzept.md), the next waves carry
the longer-roadmap items listed under
[§ "Non-Scope (explicitly not claimed)"](#non-scope-explicitly-not-claimed)
above.
